sequenceDiagram
    participant Т as Технолог
    participant API as API
    participant П as Партии
    participant БД as БД
    participant А as Аппаратчик
    participant Л as Лаборант
    
    Note over Т,А: 1. Создание и запуск партии
    
    Т->>API: Создать заказ
    API->>БД: Проверить рецептуру
    БД-->>API: OK
    API-->>Т: Заказ создан
    
    Т->>API: Запустить партию
    API->>П: StartBatch
    П->>БД: Вставить партию + шаги
    БД-->>П: Готово
    П-->>А: Уведомление
    API-->>Т: Партия запущена
    
    Note over А,БД: 2. Выполнение шагов
    
    А->>API: Начать шаг
    API->>П: StartStep
    П->>БД: Обновить статус
    БД-->>П: OK
    API-->>А: Шаг начат
    
    А->>API: Завершить шаг + параметры
    API->>П: CompleteStep
    П->>БД: Сохранить параметры
    alt Есть отклонение
        П->>API: Отклонение!
        API-->>Т: Уведомление
    end
    БД-->>П: Сохранено
    API-->>А: Шаг завершён
    
    Note over А,П: 3. Проверка завершения
    
    loop Каждый шаг
        П->>П: Есть ещё шаги?
    end
    
    alt Все шаги выполнены
        П->>БД: Статус "на контроле"
        П-->>Л: Уведомление
    end
    
    Note over Л,БД: 4. Лабораторный контроль
    
    Л->>API: Получить партию
    API->>БД: SELECT данные
    БД-->>Л: Данные партии
    
    Л->>API: Ввести результаты
    API->>БД: INSERT анализ
    
    alt Результат OK
        БД-->>Л: Одобрено
        Л-->>Т: Уведомление
    else Результат НЕ OK
        БД-->>Л: Забраковано
        Л-->>Т: Уведомление о браке
    end
